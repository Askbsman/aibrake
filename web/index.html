01Problem

AI agents burn money when they retry the same failed task without new evidence.

The 7th paid retry on the same TS2307 error rarely produces a different answer. The 4th opus call after a sonnet got stuck rarely teaches the model something it didn't already see. Most operators discover this in the monthly invoice — not before the call fires. Agent Spend Guard makes that decision before you pay.

02What it catches

Five deterministic detectors. No LLM-on-the-side. No prompt content read. Pattern matches return in milliseconds with versioned policy@ and detector@ stamps.

stale_context_retry_storm

Same failure fingerprint, growing retry count, no new files / tests / logs / diff between attempts. The canonical "7th guess" case.

same_tool_retry_loop

Same tool, same input, no diagnostic step between calls. Per-objective threshold (default 6, tunable via detector_policy).

model_escalation_without_evidence

Premium model burn — opus / GPT-5 / claude-4 — invoked on the same failure with no new evidence. Returns structured model_route if secondaryModel declared.

objective_drift

Next action falls outside objective.allowed_actions or hits objective.blocked_actions. Deterministic block.

task_budget_breach

Hard budget exceeded. The one detector that does block deterministically — your call whether to wire it as enforcing.

03How it works

Stateless Core. Stateful adapter on the agent side tracks retry history per objective. One POST, one decision.

Agent runtime
    │
    ▼  AgentActionTelemetry
CodingAgentAdapter
    │
    ▼  POST /v1/check  (Bearer asg_v1_…)
Agent Spend Guard
    │
    ▼  decision + recommended_policy + suggested_action (+ model_route)
SDK helper
    │
    ▼  guard.checkOrConfirm / checkOrDowngrade / checkShadow
Agent decides

The SDK contract has four helpers. checkShadow never blocks — it logs the decision and returns it. Recommended for the first 7 days of integration.

05Integrations

TypeScript SDK

npm spending-guard. Four helpers (check, checkShadow, checkOrConfirm, checkOrDowngrade). Structured error details.kind. Fail-open by default.

Python SDK

PyPI candidate agent-spend-guard. Stdlib-only (no requests dep). Same four helpers. err.kind / err.retryable on every error.

CodingAgentAdapter

Universal evidence model. Drop into Claude Code, Codex, Cursor, or any custom runtime. Computes failure_fingerprint, same_failure_count, evidence window for you.

x402-ready

Payment abstraction stubbed. Listing-ready for agentic.market when the hosted beta has data. No paid /v1/check yet — discipline-blocked on real partner usage.
